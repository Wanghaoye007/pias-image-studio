# 图片工作台 MVP 发布前收口

## 范围

本轮只验收图片工作台，不包含视频、短剧、数字人或其他工作台。目标链路为：

`登录 -> 创建项目 -> 上传图片 -> 放入画布 -> 选择操作 -> 配置任务 -> 提交生成 -> 进度 -> 结果 -> 继续生成 -> 采用 -> 审核或下载 -> 用量记录`

## 审计结论

### P0 核心链路阻塞

| 问题 | 收口前状态 | 本轮处理 | 结果 |
| --- | --- | --- | --- |
| 创建项目后仍停留在项目列表 | 交互断点 | 创建成功后直接打开新项目 | 已修复 |
| 工作台不能直接上传并入画布 | 只能绕到素材库 | 抽取真实上传对话框，上传后自动创建画布节点 | 已修复 |
| 空白占位节点与首张素材并存 | 画布焦点错乱 | 首张素材替换无内容占位节点 | 已修复 |
| 取消接口失败仍在前端显示已取消 | 伪成功 | 校验 HTTP 结果，供应商未确认时恢复原状态和冻结额度 | 已修复 |
| 队列恢复失败后仍可提交新任务 | 可隐藏恢复错误并重复提交 | 返回 `FAL_PERSIST_FAILED` 503，不调用上游提交 | 已修复 |
| 结果无法直观追溯原图和父结果 | 血缘只存在数据中 | 结果详情展示原始图片、版本和上一级结果 | 已修复 |
| 结果详情被选中的画布节点遮挡 | 详情不可稳定操作 | 建立独立堆叠上下文并增加浏览器点位验收 | 已修复 |
| 无核心链路集成与浏览器 E2E | 只有分散单测 | 新增域集成测试和基于真实 `dist` 的有状态 Playwright 验收 | 已修复 |

### P1 重要但不阻塞图片链路

| 问题 | 处理或当前状态 |
| --- | --- |
| 底层模型名称占据用户主信息 | 已弱化；默认展示生成、修改光影、融图、扩图、多角度、去除、抠图、超分等用户任务，技术 ID 收入折叠追踪 |

## 界面收口

- 画布是主视觉区，左侧保留素材和上传，右侧为当前任务编辑面板。
- 右侧面板从画布顶部贯穿到底部，固定为 Header / Body / Footer 三段，只有 Body 允许滚动。
- Prompt 输入区支持纵向拖拽，最小高度 112px，最大高度 48vh。
- 生成、融图、多角度、修改光影、去除、抠图、扩图、超分共用统一 Footer、48px 主按钮和任务化中文文案。
- 抠图等少参数工具明确展示“无需额外参数”，不会再由隐式 Grid 行拉伸标题或计费区域。
- 上传和导出等普通弹窗采用内容自适应高度，Header、Body、Footer 使用一致安全边距。
- 390px Sheet 使用不透明 Surface 并高于全局导航，避免底层文字透出和点击拦截。
- 结果详情使用独立顶层，画布节点不能覆盖其交互区。
