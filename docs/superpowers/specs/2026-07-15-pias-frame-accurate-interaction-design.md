# PIAS 图片工作台逐帧交互还原设计

## 1. 文档目的

本文定义 PIAS 图片工作台下一阶段的交互还原方案。实现以本地六段 TheSEA 操作录屏为行为参考，将录屏中的点击顺序、浮层位置、画布运动、节点派生、任务进度和结果组织方式转换为可自由操作、可重复执行的 PIAS 中文工作台。

还原目标是交互行为和空间关系，不复制 TheSEA 的商标、逐字文案、专有图标或演示素材。PIAS 继续使用自己的品牌、中文业务术语、企业审核和用量规则。

## 2. 已确认决策

- 采用“交互状态逐帧还原”，不采用固定脚本播放或静态截图仿制。
- 覆盖六段参考录屏：无提示词生成、人工创意、AI 合成、AI 打光、扩图与超分、视角生成。
- 每条流程都由真实点击、拖拽、缩放和参数操作触发，可重置并重复执行。
- UI 操作节奏按录屏对齐；模型生成等待压缩到 5-8 秒，但保留录屏中的状态阶段和进度表现。
- 桌面端是完整生产界面；移动端延续项目查看、任务状态和审核模式。
- 继续使用现有 React、TypeScript、React Flow 与内存领域状态，不接入真实模型或后端服务。

## 4. 还原层级

### 4.1 必须对齐

- 顶栏、左侧场景栏、悬浮工具栏、画布节点和上下文面板之间的空间关系。
- 工具打开方式、参数面板出现位置、关闭方式和提交后的收起行为。
- 节点的选中、聚焦、拖拽、缩放、分支落点与连线更新。
- 生成中的暗化预览、进度文案、进度条、百分比和完成替换。
- 一次生成多个结果时的排列、父子关系与画布自动取景。
- 打光控制点、扩图裁切网格和视角选择器等画布内控件。
- 鼠标悬停、按下、拖动、松开和禁用状态的即时反馈。

### 4.2 保持 PIAS 独立

- 品牌名称、Logo、颜色语义和导航命名。
- 工具中文名称、字段文案、错误提示和空状态。
- 素材缩略图、商品信息、场景名称和任务数据。
- 审核状态、下载门禁、用量预留与结算提示。

### 4.3 不在本期实现

- 录制视频中未出现的 TheSEA 完整产品功能。
- 真实 AI 生成、文件上传、云端保存和多人实时协作。
- 逐像素复制竞品品牌资产。
- 将录屏作为视频覆盖层伪装成交互界面。

## 5. 工作台骨架

### 5.1 固定区域

- 顶栏固定在工作区上方，承载项目路径、保存状态、撤销/重做、任务与审核入口。
- 左侧场景栏固定，宽度在展开和折叠状态间切换；场景列表与素材列表可切换。
- 工具栏悬浮于画布左上安全区，使用稳定尺寸的图标按钮与中文提示。
- 右下角保留缩放、适配全部、聚焦选中和小地图控制。

### 5.2 画布区域

- 空白拖拽平移，滚轮或触控板缩放，节点可独立拖动。
- 单击节点进入选中状态，双击进入聚焦状态，点击空白清除选中。
- 面板打开时保持目标节点可见；空间不足时自动调整面板方向并轻微平移视口。
- 创建任务后，目标占位节点立即出现，避免异步状态导致布局跳动。

### 5.3 节点视觉状态

| 状态 | 视觉表现 | 允许操作 |
| --- | --- | --- |
| 默认 | 图片为主体，标题和状态弱化 | 选中、拖拽、聚焦 |
| 悬停 | 边框提亮，快捷操作渐显 | 点击快捷动作 |
| 选中 | 明确描边与控制点 | 打开工具、继续派生 |
| 配置中 | 节点保持选中，面板贴近出现 | 修改参数、取消、提交 |
| 等待中 | 结果占位已固定 | 取消任务 |
| 生成中 | 暗化预览、进度文案与进度条 | 查看任务、取消 |
| 完成 | 预览替换，连线变为完成态 | 继续编辑、送审 |
| 失败 | 错误摘要与重试入口 | 重试、查看详情 |

## 6. 六条交互状态机

### 6.1 无提示词生成

1. 用户选择商品源节点，节点描边和快捷操作出现。
2. 用户点击“生成”，节点右侧打开场景生成面板。
3. 用户选择场景模板、比例、质量和结果数量；预计用量同步更新。
4. 用户提交后面板淡出，源节点右侧立即创建任务占位节点并建立动态连线。
5. 画布自动平移，使源节点与目标节点同时保持可见。
6. 任务依次进入排队、生成和收尾阶段，节点内进度平滑更新。
7. 完成后占位节点替换为最多四个结果，横向排列并共享批次关系。
8. 用户点击任一结果可继续派生；点击场景栏可重新定位整个分支。

### 6.2 人工创意

1. 用户从已有四图结果中选择一张，其他节点降为非焦点状态。
2. 用户拖动节点改变编排位置，连线在拖动期间实时更新。
3. 用户平移和缩放画布，视口保持惯性克制且不会误触节点拖动。
4. 用户可把外部素材拖到空白区域创建源节点，或拖到目标节点绑定为参考素材。
5. 素材节点和结果节点可并排比较；聚焦后恢复到原有画布位置。
6. 操作结束时保存节点坐标与视口，不改变结果血缘。

### 6.3 AI 合成

1. 用户选中目标场景，点击“融图”。
2. 参数面板在节点近侧出现，并显示参考素材槽位。
3. 点击素材槽位打开素材选择浮层；浮层支持分组、搜索、选中和关闭。
4. 选中参考商品后，缩略图从浮层过渡到参数槽位，浮层关闭。
5. 用户调整融合强度和补充描述后提交。
6. 目标占位节点、动态连线和生成进度按无提示词流程出现。
7. 完成后生成四个合成结果，父节点与参考素材关系均保留在任务数据中。

### 6.4 AI 打光

1. 用户选中结果节点，点击“定向光”。
2. 图片上覆盖方向控制点与边缘控制柄，参数面板同步出现。
3. 拖动控制点时光源方向预览实时变化，面板中的方向值同步更新。
4. 用户调整强度、色温和补充描述后提交。
5. 控制柄淡出，任务节点在固定位置进入生成状态。
6. 进度结束后产生打光结果分支，画布自动取景到源节点与结果。
7. 选择不同结果时，控制点使用该结果自身的最后参数初始化。

### 6.5 扩图与超分

1. 用户选中图片并点击“扩图”，节点进入带网格的构图模式。
2. 用户选择目标比例并拖动原图缩放值；网格、留白和原图边界实时变化。
3. 提交后网格收起，占位节点原位显示生成进度。
4. 完成后产生扩图结果，视口移动到新结果。
5. 用户在结果上点击“超分”，面板显示目标尺寸和细节增强级别。
6. 用户选择尺寸并提交，生成阶段复用同一进度组件。
7. 完成后产生超分结果，审核通过时才显示下载动作。

### 6.6 视角生成

1. 用户选中商品节点并点击“快速视角”。
2. 面板显示水平角度、俯仰角度和距离控制，预览随参数变化。
3. 提交后节点内显示视角生成进度，完成后得到新角度商品图。
4. 用户可从结果进入“精细视角”选择状态，面板显示候选参考角度。
5. 用户选择最多四张参考缩略图，选中状态与数量限制即时反馈。
6. 提交后生成四个视角结果，按录屏中的横向结果组排列。
7. 结果组完成后自动适配视口，但不覆盖用户随后进行的手动平移。

## 7. 动画与时间

### 7.1 UI 动画

| 动作 | 建议时长 | 曲线 |
| --- | ---: | --- |
| 悬停反馈 | 80-100 ms | ease-out |
| 按钮按下 | 60-80 ms | ease-out |
| 面板进入/离开 | 140-180 ms | ease-out / ease-in |
| 节点占位出现 | 160-220 ms | ease-out |
| 自动平移与适配 | 240-360 ms | ease-in-out |
| 结果替换 | 180-240 ms | ease-out |
| 控制柄出现/隐藏 | 120-160 ms | ease-out |

所有动画使用 `transform` 和 `opacity` 为主，避免节点尺寸变化引起连线抖动。`prefers-reduced-motion` 下取消位移动画，但保留状态变化。

### 7.2 任务时间

- 演示任务总时长为 5-8 秒，根据工具类型略有差异。
- 任务阶段统一为“排队中 → 生成中 → 优化细节 → 完成”。
- 百分比按阶段平滑增长，不在 99% 长时间停留。
- 失败和取消路径不得依赖定时器竞态；离开工作台时清理所有计时器。

## 8. 状态模型

新增前端交互状态层，不把临时浮层状态写进领域对象：

```ts
type InteractionMode =
  | 'idle'
  | 'node-selected'
  | 'configuring'
  | 'picking-asset'
  | 'editing-light'
  | 'editing-expand'
  | 'editing-angle'
  | 'submitting'

interface WorkbenchInteractionState {
  mode: InteractionMode
  selectedNodeIds: string[]
  activeTool: ToolKind | null
  anchorNodeId: string | null
  panelPlacement: 'left' | 'right'
  focusedBatchId: string | null
}
```

任务生命周期继续由领域状态管理。交互状态只决定面板、控制柄、选中和镜头行为。所有状态切换通过显式事件完成，避免多个布尔值组合出不可达状态。

## 9. 组件边界

- `Workbench`：编排领域状态、交互状态和 React Flow 视口。
- `SceneRail`：场景定位、素材选择与拖放起点。
- `ToolPalette`：工具可用性、悬停提示和激活状态。
- `ContextToolPanel`：根据工具呈现参数，不直接创建领域任务。
- `CanvasNodes`：节点状态、进度、快捷操作与画布内控制器。
- `TaskTray`：任务摘要、失败重试和取消。
- 新增 `interactionMachine`：处理显式交互事件与状态转换。
- 新增 `viewportDirector`：集中处理自动平移、适配和用户操作抢占。
- 新增工具内覆盖层：`LightOverlay`、`ExpandOverlay`、`AnglePreview`。

## 10. 错误与边界行为

- 未选择节点时，需要输入图片的工具禁用并给出中文提示。
- 面板超出视口时自动翻转；仍放不下时限制高度并内部滚动。
- 用户手动平移或缩放后，正在进行的自动取景立即停止。
- 删除源节点前检查下游结果；确认后才级联删除演示数据。
- 重复提交时只接受第一次事件，按钮进入提交中状态。
- 切换工具会清理上一个工具的覆盖层和未提交参数。
- 任务失败后保留节点位置，重试复用原占位节点。

## 11. 测试策略

### 11.1 单元测试

- 每个交互事件只进入一个合法状态。
- 关闭、切换工具、任务提交和重置均能回到预期状态。
- 面板放置、任务阶段、进度和结果组布局为确定性输出。
- 用户视口操作能够抢占自动取景。

### 11.2 组件与流程测试

- 六条流程分别覆盖从选择源节点到结果完成的完整路径。
- 素材选择、控制柄、扩图网格和视角参考选择支持键盘与鼠标操作。
- 任务取消、失败、重试和重复提交均有覆盖。
- 审核未通过时下载入口保持关闭。

### 11.3 视觉验收

- 以 1440 x 900 为主验收视口，并补测 1920 x 1080、1280 x 800、1024 x 768 和 375 x 812。
- 对每段录屏建立关键帧检查点，使用截图差异与人工复核共同验收。
- 重点检查面板锚点、节点尺寸、分支间距、连线曲率、控制柄位置和画布取景。
- 所有桌面视口不得出现控件重叠、文字截断、空白画布或节点越出不可恢复区域。

## 12. 验收标准

1. 六段录屏对应的主流程都能从当前工作台独立触发并完整走通。
2. 每个录屏关键帧都有可复现的 UI 状态和自动化截图。
3. 用户可以自由改变操作顺序，界面不会退化为固定演示脚本。
4. 节点拖拽、画布平移、缩放和自动取景之间不存在明显跳动或争抢。
5. 任务占位、进度、完成、失败、取消和重试状态完整。
6. 所有可见文案为中文，PIAS 品牌与企业业务规则保持不变。
7. 自动化测试、生产构建和桌面/移动浏览器视觉检查全部通过。
