# 图片工作台设计系统收口规范

## 状态

- 日期：2026-07-23
- 范围：图片工作台右侧工具编辑器、工作台内弹窗及其共享控件
- 决策：采用方案 A。右侧工具编辑器保持全高停靠；普通 Modal 随内容自适应高度
- 约束：不新增业务功能，不修改任务参数、Fal 工作流、额度、审核、导出或数据持久化逻辑

## 背景与问题

当前工具编辑器由同一个 `ContextToolPanel` 承载八类图片任务，但组件没有明确的 Header、Body、Footer 三段结构。全高 Grid 中的普通工具依赖隐式行布局，抠图等少参数工具只渲染标题和 Footer 时，空余高度会被错误分配到隐式行，导致标题区和 Footer 被拉伸、操作按钮远离内容、滚动时机异常。

视觉样式还存在结构性级联风险：`soft-glass.css` 中多轮设计规则重复定义 `.context-panel`、Footer、控件和高级编辑器。后写规则能够改变前面规则的几何语义，现有静态 CSS 断言无法证明浏览器最终布局正确。

本轮目标不是继续追加局部覆盖，而是建立可复用的布局和组件合同，并让八类工具在同一套合同下保持功能完整。

## 设计原则

1. 画布优先。工具编辑器停靠在画布右侧，打开时画布为其预留空间，节点和画布控制不得被遮挡。
2. 内容不拉伸。面板可以全高，但参数内容始终从顶部开始排列，少参数工具不会为了填满高度而放大间距或区块。
3. 单一滚动容器。Header 和 Footer 固定，只有 Body 可以纵向滚动；素材选择浮层除外。
4. 组件表达行为。布局由 `PanelShell`、`PanelBody` 和 `GenerationFooter` 等组件语义决定，不能依赖 DOM 顺序和隐式 Grid 行碰巧成立。
5. 控件一致、任务差异清楚。输出数量、比例、点数和主操作复用同一组件；各工具只负责自己的参数内容。
6. 状态真实。Hover、Focus、Active、Disabled、Submitting、Error 必须可辨认，不能隐藏错误或用静态成功状态替代真实交互。
7. 减少嵌套卡片。区块优先使用间距和分隔线组织，只有确实需要边界的选择器、素材槽和状态提示使用 Section Surface。

## 设计令牌

### 间距

组件只能使用以下间距级别：

| Token | 数值 | 用途 |
| --- | ---: | --- |
| `space-1` | 4px | 图标与微型状态间隔 |
| `space-2` | 8px | 紧凑控件内部间隔 |
| `space-3` | 12px | 表单标签与控件间隔 |
| `space-4` | 16px | 相邻控件、Section 内边距 |
| `space-6` | 24px | 面板安全区、Section 间距、Footer 底部 |
| `space-8` | 32px | 大区块分隔 |
| `space-12` | 48px | 仅用于空状态或大型结构分隔 |

禁止为单个页面引入 18px、20px、22px、28px 等新的孤立间距。现有值在迁移时映射到最接近的令牌。

### Surface

| 层级 | Token | 建议值 | 用途 |
| --- | --- | --- | --- |
| 0 | `surface-canvas` | `#0b0e12` | 无限画布和页面底层 |
| 1 | `surface-panel` | `#15191f` | 停靠面板、Modal 主体 |
| 2 | `surface-section` | `#1b2028` | 素材槽、分段选择器、状态区 |
| 3 | `surface-raised` | `#242a34` | 选中项、菜单、悬浮内容 |

Glass 高光和模糊只作为材质增强，不得承担层级区分。关闭 `backdrop-filter` 后，四级 Surface 仍须清晰可辨。

### 圆角

| Token | 数值 | 用途 |
| --- | ---: | --- |
| `radius-sm` | 8px | 小型输入、菜单项 |
| `radius-control` | 10px | 按钮、输入框、选择器 |
| `radius-section` | 12px | 素材槽、参数区、状态区 |
| `radius-modal` | 16px | 独立 Modal |
| `radius-pill` | 999px | 状态徽标、真正的 Pill 控件 |

停靠面板与画布相接的外边缘保持 0 圆角。不能用超大圆角把普通矩形按钮伪装成 Pill。

### 字体层级

| 角色 | 字号 / 行高 | 字重 |
| --- | --- | --- |
| 辅助标题 | 12px / 18px | 500 |
| 控件标签 | 13px / 20px | 500 |
| 正文 | 14px / 22px | 450 |
| Section 标题 | 15px / 22px | 600 |
| 工具标题 | 24px / 32px | 650 |

工具标题和“图片处理”辅助标题必须形成清晰层级。标题区域不得使用会裁切中文笔画的固定高度或过紧行高。

### 控件尺寸

| 尺寸 | 高度 | 用途 |
| --- | ---: | --- |
| SM | 40px | 图标按钮、紧凑选择器、次操作 |
| MD | 48px | 输入、分段控件、主要按钮 |

工作台工具面板不再使用 42px、44px、46px、52px、56px 等混合高度。Slider 轨道可低于 40px，但其完整点击区域不得小于 40px。

## 布局合同

### 停靠工具编辑器

桌面端工具编辑器使用固定三段布局：

```text
PanelShell
├── PanelHeader       auto
├── PanelBody         minmax(0, 1fr), overflow-y: auto
└── GenerationFooter  auto
```

- Panel 宽度：`396px`（大于等于 1440px）、`360px`（900px 至 1439px）、`320px`（768px 至 899px）。
- Panel 高度：画布内容区从顶部到底部全高，不包含全局导航。
- Header：最小高度 72px，左右安全边距 24px，下边框 1px。
- Body：上下左右 24px，内容使用 `align-content: start`，Section 间距 24px。
- Footer：顶部 16px、左右 24px、底部 24px；主按钮高度 48px。
- Header 和 Footer 不参与滚动，不允许被 Body 内容推离可视区域。
- Body 只有在内容真实超过可用高度时才出现滚动条。
- Prompt 默认最小高度 112px，可纵向拖拽，最大高度不能遮住 Footer。
- Header 内所有文字与关闭按钮的包围盒必须处于面板安全区，不能与边界或圆角相交。

### 普通 Modal

- 默认宽度由内容类型决定，常规表单最大宽度 520px。
- 高度使用内容自适应，不设置视口百分比最小高度。
- 最大高度为 `calc(100dvh - 48px)`；只有超过该值时，Modal Body 滚动。
- Header 底部 24px，Section 间距 24px，卡片间距 16px，Footer 顶部 24px。
- Footer 不使用绝对定位，不制造内容覆盖。

### 窄屏和移动端

- 768px 至 899px 使用 320px 停靠面板，Body 仍为唯一滚动容器。
- 小于 768px 时，工具编辑器转为全屏 Sheet，Header 和 Footer 固定，Body 滚动。
- 任何工具不得仅通过 `display: none` 在移动端消失。若核心画布编辑能力不适合触摸设备，应显示明确的受限状态，而不是隐藏入口或按钮。

## 组件架构

### `ToolPanelShell`

负责宽度、三段 Grid、安全区、滚动边界、关闭行为、标题层级和响应式形态。它不读取任务参数，也不执行生成。

接口至少包含：

- `title`
- `eyebrow`
- `toolId`
- `initialFocusRef`
- `onClose`
- `body`
- `footer`

### `PanelBody`

负责顶部对齐、Section 间距和唯一滚动容器。少参数工具也必须渲染 Body。抠图 Body 显示简洁的“无需额外参数”状态和当前源图说明，不增加新的业务配置。

### `GenerationFooter`

八类工具共享同一 Footer 框架，按能力显示插槽：

- `resetAction`
- `outputSelector`
- `aspectRatioControl`
- `creditEstimate`
- `validationMessage`
- `primaryAction`

Footer 的布局分为信息行和操作行。隐藏某个插槽不能改变主按钮高度、底部安全区或其他控件的视觉样式。

### 共享控件

- `OutputSelector`：统一 1、2、4 的选择状态和键盘行为。
- `AspectRatioControl`：统一比例选项、标签和高度。
- `CreditEstimate`：紧凑信息展示，不使用占据大面积的独立卡片。
- `PanelSection`：使用标题、内容和可选分隔线，不允许 Section 内再嵌套装饰卡片。
- `RangeControl`：统一标签、数值、轨道和可点击高度。
- `SegmentedControl`：统一选中、悬停、焦点和禁用状态。

## 八类工具映射

| 工具 | Body 内容 | Footer 内容 |
| --- | --- | --- |
| 生成 | 场景模板、质量、Prompt | 输出、比例、点数、开始生成 |
| 融图 | 参考素材、商品位置 | 输出、比例、点数、开始融图 |
| 多角度 | 预览、镜头参数、风险提示 | 重置、输出、比例、点数、生成视角 |
| 修改光影 | 预览、光源参数、风险提示 | 重置、输出、点数、生成光影修改 |
| 去除 | 笔刷大小、蒙版状态 | 校验提示、点数、开始去除 |
| 抠图 | 无额外参数状态、当前源图说明 | 点数、开始抠图 |
| 扩图 | 锚点、缩放、Prompt | 输出、比例、点数、开始扩图 |
| 超分 | 目标尺寸、细节增强 | 点数、开始超分 |

现有能力之外不新增参数。不同工具的差异通过能力配置和 Footer 插槽表达，不能复制新的 Footer 实现。

## 交互合同

### 打开与关闭

- 工具按钮打开编辑器后，选中源节点必须完整位于面板左侧。
- 初始焦点进入该工具第一个可操作参数；没有参数的抠图聚焦主操作按钮。
- `Escape` 关闭当前最上层浮层；素材选择打开时先关闭素材选择，再关闭工具编辑器。
- 关闭后焦点返回原工具按钮。

### 参数控件

- Button、Segmented、Switch 和 Slider 都必须响应指针与键盘操作。
- 选中状态使用 `aria-pressed` 或控件原生状态，并与视觉状态一致。
- Slider 的视觉数值与提交参数保持一致。
- Reset 只恢复当前工具参数和 Prompt，不改变选中节点、项目或其他工具状态。

### 提交

- 主按钮始终位于 Footer 操作行，在支持的桌面和窄屏尺寸下无需滚动即可看到。
- 提交中显示真实 `Submitting` 状态并禁用重复提交。
- 去除未提供蒙版、融图未提供参考素材、额度不足时必须显示可理解的禁用原因。
- 点击主按钮必须走现有 `onRun`、任务队列和错误处理流程，不能引入静态成功状态。

### Motion

- 面板进入 180ms 至 220ms，使用位移不超过 8px 的淡入。
- Button Active 使用轻微 `scale(.98)` 或 1px 位移，不能改变布局尺寸。
- `prefers-reduced-motion` 下取消非必要动效。

## CSS 架构与迁移

1. `design-tokens.css` 保留全局语义 Token，并补齐本规范中的 Spacing、Surface、Radius、Type 和 Control Size。
2. 新建独立的工作台面板样式模块，集中维护 Tool Panel、Footer 和共享控件规则。
3. 从 `soft-glass.css` 删除被迁移组件的重复定义，禁止在文件末尾继续追加兼容层。
4. `ContextToolPanel` 退化为任务能力编排器；布局职责迁移到共享组件。
5. `AdvancedEditorFooter` 合并进共享 `GenerationFooter`，光影和多角度只保留各自 Body 编辑器。
6. 每迁移一类工具就运行该工具的组件测试和浏览器几何测试，避免一次性替换后难以定位回归。

## 测试与验收

### 组件测试

八类工具分别验证：

- 正确标题和 Body 内容
- 共享 Footer 插槽
- 参数变更回调
- 输出和比例选择
- Reset 行为
- 禁用原因
- Close、Escape 和焦点返回
- 主按钮仅触发一次真实 `onRun`

### 浏览器几何测试

在 1440x960、1024x768、768x900 和 390x844 验证：

- 标题和关闭按钮距离面板左右边界至少 24px
- 标题文字不与边界、圆角或按钮相交
- Header、Body、Footer 三段不重叠
- Footer 底边与面板底边间距为 24px
- 主按钮高度为 48px且完整可见
- 少参数工具 Body 不产生滚动条
- 长参数工具只允许 Body 滚动
- 打开面板后源节点不被遮挡
- 页面无横向溢出

### 交互 E2E

对八类工具执行参数操作矩阵，至少覆盖：

- 打开、关闭、切换工具
- 修改每一种控件类型
- 提交、重复点击保护、失败和重试
- 去除蒙版校验、融图素材选择
- 多角度和光影重置
- 页面刷新后的任务恢复

E2E 必须验证 API 或状态变化，不能只验证按钮存在。

### 视觉证据

每种工具至少保留一张桌面稳定帧。额外保留：

- 抠图少内容面板
- 生成长 Prompt 面板
- 多角度高级编辑器
- 1024x768 Body 滚动状态
- 390x844 全屏 Sheet

## 完成标准

- 八类工具全部迁移到统一三段布局和共享 Footer。
- 截图中不存在标题裁切、异常大空白、过早滚动、Footer 贴边、左右 Padding 不一致或控件高度混用。
- 所有原有按钮、参数、提交和键盘交互通过回归测试。
- `soft-glass.css` 不再承担工具面板的重复布局规则，且不存在新的末尾兼容覆盖层。
- `repo:check`、`typecheck`、`lint`、全量测试、构建、图片 MVP E2E、Acceptance 与 Release Preflight 均按真实结果执行并记录。

## 风险与控制

- **级联删除风险**：迁移时按组件选择器逐段删除，并用浏览器计算样式和截图验证，避免误伤画布节点或其他弹窗。
- **高级编辑器风险**：光影和多角度保留参数逻辑，只替换外壳和 Footer；提交输入需要快照测试。
- **响应式风险**：桌面、窄屏和移动端分别建立几何断言，不能只依赖单一 1440px 截图。
- **功能回归风险**：每个视觉迁移提交都必须同时包含该工具的交互测试，不接受纯截图通过。
