# PIAS 图片工作台 UI 设计指南针

## 1. 设计定位

### 方向名称

**Soft Industrial Glass / 柔性工业玻璃**

PIAS 不是展示型网站，也不是消费级图片 App。它是面向高频内容生产的节点式工作台，因此视觉升级必须同时满足：长时间使用不疲劳、复杂流程可扫描、操作反馈有手感、品牌与素材始终比 UI 装饰更突出。

### 一句话原则

**让工具像玻璃一样浮在内容之上，让节点像可靠的生产对象一样落在画布之中。**

### 推荐方案与备选

| 方案 | 表现 | 取舍 | 结论 |
| --- | --- | --- | --- |
| A. 克制高级玻璃 | 深色中性画布、分级圆角、局部毛玻璃、柔和弹性反馈 | 兼顾高级感、密度与性能 | **采用** |
| B. 强 Liquid Glass | 高透明、高模糊、明显折射与悬浮 | 展示感强，但长期使用分心，文字对比不稳定 | 不采用 |
| C. 轻量企业圆润 | 实色面板、统一圆角、轻阴影 | 风险最低，但不足以形成新版辨识度 | 仅作为低性能回退 |

## 2. 灵感来源与转译

这套方向参考成熟产品的原则，不复制其皮肤。

- [Apple Human Interface Guidelines: Materials](https://developer.apple.com/design/human-interface-guidelines/materials)：玻璃应主要服务导航、工具和临时控制层，不应覆盖内容层；玻璃使用必须克制，并提供降低透明度的适配。
- [Linear: A calmer interface for a product in motion](https://linear.app/now/behind-the-latest-design-refresh)：高密度产品中，主任务必须占据最强视觉权重，辅助导航应主动后退；减少边框噪声比增加装饰更高级。
- [Microsoft Fluent Acrylic](https://learn.microsoft.com/en-us/windows/apps/design/style/acrylic)：毛玻璃由透明度、模糊、色调、细噪点和边缘分离共同构成；避免多层 Acrylic 相邻或叠加。
- [Windows Motion](https://learn.microsoft.com/en-us/windows/apps/design/signature-experiences/motion)：动效应直接、连贯、上下文相关；相同入口的面板必须使用一致的进入方向、时长和缓动。
- [n8n 工作区结构](https://support.n8n.io/article/how-to-access-the-n-8-n-admin-panel)：节点工具的核心结构是画布、参数面板和执行记录。PIAS 保留这三个稳定区域，不因视觉升级改变操作心智。
- [Motion Spring](https://motion.dev/docs/spring)：弹性只用于短距离位移和状态切换，不用于持续漂浮或大面积装饰。
- [WCAG 2.2 动效指南](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions)：所有非必要位移动效支持 `prefers-reduced-motion`，避免自动循环和视差运动。

## 3. 六条设计原则

### 3.1 Canvas First

画布、商品图和节点链路永远是第一视觉层。导航、素材栏、工具栏和任务抽屉降低亮度与对比，只有在激活时上浮。

### 3.2 Glass Is Functional

玻璃只用于悬浮工具栏、参数面板、弹窗、任务抽屉、下拉菜单和拖拽反馈。场景节点、结果节点和大面积内容区使用高不透明表面，保证图片与文字稳定。

### 3.3 Soft Geometry, Clear Hierarchy

圆润不等于所有元素都做成胶囊。尺寸越大、离画布越高，圆角越大；越接近精密控制，圆角越小。状态标签和开关可以使用胶囊形，命令按钮保持清晰的工具形态。

### 3.4 One Accent, Semantic Color

蓝色只表示选中、主操作和可连接状态。绿色、黄色、红色仅表达成功、警告、失败。不要使用紫蓝渐变，不用多彩光晕制造“AI 感”。

### 3.5 Motion Explains State

动效解释“从哪里来、发生了什么、去了哪里”。节点拖拽必须直接跟手；参数面板从对应节点方向出现；任务完成通过一次短促的边缘高光反馈，而不是持续闪烁。

### 3.6 Premium Without Fragility

去掉阴影、模糊或动效后，层级仍然成立。玻璃效果是增强，不是结构本身；低性能、降低透明度和降低动态模式都有可靠实色回退。

## 4. 视觉系统

### 4.1 配色方案：Graphite Mineral

V1 采用单一深色主题 **Graphite Mineral / 石墨矿物**。比例约为 80% 深色中性、15% 抬升表面、5% 强调与语义色。丰富颜色应主要来自商品图片，不由 UI 装饰制造。避免纯黑与单一蓝灰，使用略带冷暖差异的石墨中性层形成空间深度。

背景允许极弱的冷暖径向光差来增强玻璃可见性，但不得出现独立光球、紫色渐变或抢眼装饰。V1 不同时实施浅色主题，避免在视觉系统尚未稳定前扩大验证范围。

### 4.2 材质层级

| 层级 | 材质 | 规则 |
| --- | --- | --- |
| L0 画布 | 实色 + 低对比网格 | 不模糊，不发光 |
| L1 内容 | 94%-98% 不透明表面 | 节点、素材项、结果内容，保证图片和文本稳定 |
| L2 常驻工具 | 72%-84% 不透明玻璃 | 导航、素材栏、顶部状态栏，模糊 14-18px |
| L3 临时工具 | 56%-72% 不透明玻璃 | 参数面板、菜单、任务抽屉，模糊 18-24px |
| L4 阻断层 | 高不透明玻璃 + 遮罩 | 弹窗和审核确认，背景降低亮度而非强模糊 |

标准玻璃配方：背景色调 + `backdrop-filter: blur(...) saturate(115%)` + 1px 半透明边缘 + 上沿内高光 + 低透明噪点。禁止玻璃套玻璃，禁止相邻面板使用不同模糊强度形成接缝。

| Token | 尺寸 | 用途 |
| --- | --- | --- |
| `--radius-xs` | 6px | 微型控制点、分段选项内部元素 |
| `--radius-sm` | 8px | 紧凑标签、状态元素、结果快捷操作 |
| `--radius-control` | 10px | 按钮、输入框、分段控制、导航按钮 |
| `--radius-md` | 14px | 节点、素材项、数据卡片、工具组 |
| `--radius-lg` | 16px | 参数面板、任务抽屉、菜单、工具浮层 |
| `--radius-xl` | 20px | 模态弹窗和大型浮层 |
| `--radius-pill` | 999px | 状态标签、开关、进度徽标，仅限语义成立时 |

同一容器内，子元素圆角必须小于父元素至少 4px。图片与节点共用顶部圆角，节点底部信息区不做第二张嵌套卡片。

节点悬停只提升边缘亮度和接触阴影，不大幅漂浮。节点选中使用 1px 强调边与 `--ring-selected`，不使用霓虹光。禁止四周等强阴影、连续发光和多个彩色阴影叠加。

### 4.5 字体与排版

- 字体栈：`"Manrope Variable", "Noto Sans SC Variable", "Noto Sans SC", sans-serif`。通过本地字体包或构建产物加载，不依赖运行时远程字体。
- 拉丁、数字和英文模型名由 Manrope 优先渲染；中文由 Noto Sans SC 渲染，保持相近的 x-height 与字重观感。
- 仅使用 400 / 500 / 600 三档字重。600 仅用于页面标题、节点标题、关键数值和主操作。
- 数字、额度、尺寸、Seed、任务编号统一使用 `font-variant-numeric: tabular-nums`。
- 所有字距为 0，不使用负字距，不随视口宽度缩放字号。

| Token | 字号 / 行高 | 字重 | 用途 |
| --- | --- | --- | --- |
| `--type-caption` | 11px / 16px | 500 | 桌面状态、短标签、模型辅助信息 |
| `--type-label` | 12px / 18px | 500 | 控件标签、按钮、表头、导航 Tooltip |
| `--type-body-sm` | 13px / 19px | 400 | 工作台高密度正文、节点元数据 |
| `--type-body` | 14px / 21px | 400 | 参数说明、弹窗正文、移动端正文 |
| `--type-title-sm` | 15px / 22px | 600 | 节点标题、面板标题、列表主文字 |
| `--type-title` | 18px / 26px | 600 | 页面分区标题、弹窗标题 |
| `--type-heading` | 24px / 32px | 600 | 二级页面 H1 |
| `--type-display` | 32px / 40px | 600 | 仪表盘关键总览，仅少量使用 |

节点标题单行省略，辅助信息最多两行。按钮不允许因为 Loading 文案切换而改变宽度；表格数字列固定使用等宽数字对齐。

### 4.6 间距体系

全局采用 **4px 基础网格**。2px 和 6px 仅用于图标与文字的光学校正，不作为容器主间距。

| Token | 尺寸 | 用途 |
| --- | --- | --- |
| `--space-0` | 0 | 无间距 |
| `--space-1` | 4px | 图标内部、紧密状态 |
| `--space-2` | 8px | 图标与文字、小控件间距 |
| `--space-3` | 12px | 按钮横向 Padding、紧凑列表 |
| `--space-4` | 16px | 面板内边距、字段组间距 |
| `--space-5` | 20px | 卡片内容和桌面区块间距 |
| `--space-6` | 24px | 页面区块、弹窗主要分区 |
| `--space-8` | 32px | 页面大区块和桌面页边距 |
| `--space-10` | 40px | 空状态与大型区域 |
| `--space-12` | 48px | 二级页面纵向节奏 |
| `--space-16` | 64px | 页面级留白，工作台内很少使用 |

#### 间距组合规则

- 图标按钮：图标 16-18px，点击区 36-40px；移动端点击区不低于 44px。
- 普通按钮：高度 36px，横向内边距 12px；主 CTA 高度 40px，横向内边距 16px。
- 输入控件：高度 38px，左右内边距 12px；Textarea 最小高度 88px，内边距 12px。
- 面板：桌面内边距 16px；大型弹窗 20-24px；移动端边距 16px。
- 字段：标签与控件间距 6px；同组控件间距 8px；字段组之间 16px。
- 列表：紧凑行 40px，标准行 48px，带图片行 64-80px。
- 页面：桌面外边距 32px、区块间距 32px；平板 24px；移动端 16px。

### 4.7 布局与尺寸

| 区域 | 桌面 | 平板 | 移动端 |
| --- | --- | --- | --- |
| 主导航 | 60px 固定轨道 | 56px 固定轨道 | 顶部 52px，7 列图标导航 |
| 工作台顶部状态栏 | 52px | 52px | 48px，状态自动收敛 |
| 场景素材栏 | 248px，可折叠为 48px | 224px，可折叠 | 不进入编辑态，使用移动预览 |
| 参数面板 | 336px，最大高为画布减 32px | 320px | 100% 宽底部面板，仅单层显示 |
| 结果详情 | 360px | 336px | 全屏面板 |
| 模态弹窗 | 420-720px，按任务固定宽度 | `min(92vw, 640px)` | 贴底或全屏，不保留悬空窄卡片 |
| 二级页面内容 | 最大 1440px，左右 32px | 左右 24px | 左右 16px |

桌面断点为 1200px、平板断点为 900px、移动编辑边界为 768px、小屏修正为 420px。固定格式元素必须使用稳定宽高或 `aspect-ratio`，避免图标、Loading 和长文字触发布局跳动。

### 4.8 图标体系

- 统一使用 Lucide，不混用手绘 SVG。
- 16px 用于表格、输入框和辅助操作；18px 用于常规工具；20px 用于导航和主动作；24px 仅用于空状态。
- 默认描边 1.75；激活图标 2；禁用态只降低颜色，不缩小图标。
- 纯图标按钮必须有 Tooltip 和 `aria-label`。有成熟符号时不使用文字胶囊代替，例如关闭、下载、复制、缩放、收藏。

### 4.9 交互状态矩阵

所有交互组件至少提供以下状态，禁止只设计 Default 和 Hover。

| 状态 | 视觉规则 |
| --- | --- |
| Default | 使用默认表面、文字与 10% 边缘 |
| Hover | 表面提高一级，边缘升至 16%，上移不超过 1px |
| Pressed | `scale(.98)` 或下移 1px，亮度短暂降低，80-120ms 内反馈 |
| Selected | 弱强调背景 + 强调色前景；需要定位时增加 1px 强调边 |
| Focus Visible | 2px 外置焦点环，与 Selected 可同时存在 |
| Disabled | 文字和图标降至 40%-48%，保留布局，移除阴影与位移 |
| Loading | 保持原尺寸，替换局部图标或进度，不改变主文案宽度 |
| Success | 一次 200-300ms 高光，随后回到静态成功色 |
| Warning / Error | 弱语义背景 + 语义色图标和文字，不整卡高饱和 |
| Dragging | 关闭位置过渡，阴影收紧，缩放不超过 1.02，连接点增强 |

### 4.10 Token 使用纪律

- 组件 CSS 必须通过颜色、间距、圆角、阴影、字体和动效 Token 组合，不直接重复写十六进制颜色或同义数值。
- 语义色按 `--color-{semantic}-foreground`、`--color-{semantic}-solid`、`--color-{semantic}-solid-hover`、`--color-{semantic}-subtle` 命名。
- 1px 边缘、2px 光学校正、图标描边和 React Flow 连线宽度允许作为 Token 之外的例外；容器 Padding、Gap、圆角和字号不允许随意例外。
- 组件通过类名或 `data-state` 表达 `selected`、`loading`、`error`、`dragging` 等状态，不在 JSX 中拼接任意颜色值。
- `--color-primary-solid`、`--color-success-solid`、`--color-warning-solid`、`--color-danger-solid` 与 `--text-primary` 的对比分别约为 4.79:1、5.04:1、4.92:1、4.75:1。
- 新组件进入代码前必须先在本规范找到对应的组件族；没有对应族时，先扩展规范再实现。

## 5. 全组件设计规格

### 5.1 全局框架与导航

## 6. 动效指南

### 6.1 时间尺度

| 类型 | 时长 | 缓动 |
| --- | --- | --- |
| Hover / Focus | 90-120ms | `ease-out` |
| Press / Toggle | 80-140ms | 快速回弹，视觉缩放最低 0.97 |
| Tooltip / Menu | 140-180ms | 淡入 + 4px 位移 |
| 参数面板 / 抽屉 | 180-240ms | `cubic-bezier(.2,.8,.2,1)` |
| 节点吸附 / 结果出现 | 240-320ms | 低弹性 Spring，禁止明显来回抖动 |
| 画布聚焦 / Fit View | 260-360ms | 平滑 ease-in-out |

### 6.2 手感规则

- **Hover 是确认可操作，不是表演。** 只改变边缘、亮度和 1px 位移。
- **Pressed 必须立即发生。** 按下 60ms 内出现压缩或亮度变化，释放后回弹。
- **Drag 必须零延迟。** 拖拽时关闭节点阴影和位置过渡，只保留轻微缩放与抓取态。
- **Panel 必须有来源。** 左侧触发的面板从左进入，节点右侧参数从右进入。
- **Success 只庆祝一次。** 结果出现可有一次 300ms 边缘扫光，之后回到静态。
- **Loading 不推动布局。** 骨架、进度条、错误文字都在稳定尺寸内变化。

### 6.3 禁止动效

- 不使用自动循环漂浮、呼吸、旋转或背景流光。
- 不使用大范围视差和跟随鼠标的背景光球。
- 不给所有图标添加独立动画。
- 不在拖拽、缩放和画布平移中使用高成本模糊动画。

## 7. 可访问性与性能

- 所有文本在玻璃上仍满足 WCAG AA 对比；玻璃后方图片过亮时自动提高遮罩不透明度。
- `prefers-reduced-motion: reduce` 下关闭位移、缩放和弹性，仅保留 80-120ms 的颜色或透明度变化。
- `prefers-contrast: more` 或透明度关闭时，所有玻璃回退为 `--surface-solid`，边框提高到 16%-22% 白色透明度。
- 同屏高模糊层控制在 5 个以内；大面积长驻面板优先 14px 模糊，临时小面板才使用 20-24px。
- 动效优先使用 `transform` 和 `opacity`，不动画 `width`、`height`、`box-shadow` 的大范围模糊半径。
- 移动端降低阴影半径、模糊强度和同时显示的浮层数量。

## 8. 明确禁区

1. 不把整个页面做成半透明卡片拼贴。
2. 不在页面章节外再包“浮动卡片”。
3. 不使用紫色、紫蓝渐变或霓虹光作为 AI 身份表达。
4. 不给所有元素相同的大圆角；胶囊仅用于状态和开关。
5. 不使用玻璃套玻璃、卡片套卡片。
6. 不让毛玻璃降低商品图、标签文字和审核状态的可读性。
7. 不因视觉升级改变已有节点拖拽、连接、参数和任务流程。
8. 不使用无限循环动效制造“活力”。

## 9. 实施顺序

### Phase 1：基础设计令牌

建立颜色、材质、圆角、阴影、时长、缓动与降级模式；替换分散的硬编码值。

### Phase 2：全局框架与二级页面

升级主导航、全局页头、首页 KPI、项目、素材、审核、用量、企业管理、表格与列表，建立页面级视觉基线。

### Phase 3：工作台骨架与表单

升级顶部状态栏、素材栏、画布工具栏、参数面板、任务抽屉、菜单和所有输入控件，形成稳定的材质与间距层级。

### Phase 4：节点、结果与生产反馈

统一节点 hover / selected / dragging / running / succeeded 状态，补全图像覆盖层、结果决策、对比、详情、导出与审核反馈。

### Phase 5：动效、响应式与回退

统一进入方向和时间尺度，增加 reduced-motion、低透明度和高对比度回退，并在桌面、平板和移动端截图复查。

## 10. 验收标准

- 第一眼焦点落在画布商品与当前节点，而不是导航或毛玻璃效果。
- 截图转为灰度后，画布、节点、工具和临时面板仍有清晰的四级层次。
- 圆角有明确层级，页面中不存在“所有东西都像胶囊”的观感。
- 同一类面板的进入方向、时长和关闭行为一致。
- 按钮按下、节点选中、拖拽开始、任务完成都有可感知且不拖沓的反馈。
