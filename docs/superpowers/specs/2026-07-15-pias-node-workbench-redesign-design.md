# PIAS 中文节点式图片工作台改版设计

## 1. 文档目的

本文定义 PIAS 图片 MVP 的工作台改版方案。改版以本地 TheSEA 操作视频中呈现的画布式生产流程为交互参考，将现有固定三栏图片工作室升级为中文、画布优先、节点可自由拖拽的企业级内容生产工作台。

参考对象仅限界面布局、空间组织和交互范式。PIAS 保留自己的品牌、业务术语、权限、审核、用量和素材数据，不复制竞品商标、文案或专有视觉资产。

## 2. 目标与非目标

### 2.1 目标

- 图片工作台以全屏无限画布为核心，支持平移、缩放和节点自由拖拽。
- 原图、生成任务和结果以可视化节点呈现，并通过连线形成可追溯的版本分支。
- 生成、融图、快速视角、定向光、去除、抠图、扩图和超分通过悬浮工具栏进入。
- 所有编辑参数在当前节点附近的上下文浮层中完成，减少跨区域视线移动。
- 全站可见文案统一为中文，并保留企业审核、用量预留与结算、下载门禁等 MVP 规则。
- 保持现有 React、TypeScript、React Flow 和领域状态模型，不引入新的大型前端框架。

### 2.2 非目标

- 本次不接入真实图片生成模型、对象存储、账号系统或后端任务队列。
- 不实现多人实时协同、画布评论、复杂时间轴或视频生产能力。
- 不复制 TheSEA 的名称、Logo、图标素材、示例数据或逐字文案。
- 移动端不提供完整节点编辑，只提供项目预览、任务状态和审核操作。

## 3. 视频分析结论

六段操作视频共同呈现出以下稳定模式：

- 工作区是近黑色全屏无限画布，带低对比度点阵背景。
- 左侧固定窄栏同时承担场景树和素材库，主体不使用传统表单页或大卡片区。
- 工具以紧凑的竖向图标栏悬浮在画布左上方。
- 场景节点以图片为主体，标题、状态和操作保持弱化。
- 衍生结果按空间关系落在源节点附近，并用浅色曲线连接。
- 设置、素材选择和任务参数均以深色浮层出现在当前节点周围。
- 生成进度直接出现在目标节点内部，节点位置在执行期间保持稳定。
- 定向光、扩图和视角编辑把控制柄直接覆盖在图片上，而不是放进独立表单页。
- 右下角集中放置缩放、适配画布和视图控制。

这些模式将被转化为 PIAS 的节点式生产语言，并与 PRD 中的素材、任务、审核和用量规则结合。

## 4. 信息架构

### 4.1 全局导航

桌面端保留 56px 宽的全局导航栏，使用图标加悬浮提示：

- 首页
- 项目
- 图片工作台
- 素材库
- 审核
- 用量
- 企业管理

当前模块使用蓝色指示条和高对比度图标，不使用大面积背景色块。工作台页面默认折叠非必要全局信息，使画布获得最大面积。

### 4.2 工作台顶栏

顶栏高度为 48px，包含：

- 项目路径与当前项目名称
- 自动保存状态
- 撤销与重做
- 任务队列入口
- 审核状态
- 用户菜单

顶栏不放置营销说明、统计卡片或大标题。

### 4.3 左侧场景与素材栏

左侧面板默认宽 224px，可折叠为 44px。面板分为上下两个区域：

- 场景：展示场景、版本、来源和审核状态，可点击定位到画布节点。
- 素材库：使用“商品 / 上传 / 品牌”三个页签，提供搜索和缩略图拖入画布。

素材拖入空白画布时创建源图片节点；拖入已选节点时作为参考素材绑定，不自动触发任务。

### 4.4 无限画布

画布占据剩余可用空间，背景为近黑色点阵。核心操作包括：

- 鼠标拖拽空白处平移画布。
- 滚轮或触控板缩放画布。
- 节点可自由拖拽，位置写回前端状态。
- 单击选中节点，多选用于批量移动与比较。
- 双击节点进入聚焦编辑模式。
- 右下角提供缩放比例、适配全部、聚焦选中和小地图开关。

## 5. 节点体系

### 5.1 源图片节点

源图片节点表示上传素材、商品图或历史结果。节点包括：

- 场景名称
- 图片预览
- 素材来源标记
- 审核状态点
- 选中时出现的快捷动作

节点以图片为视觉主体，宽度默认为 240px，圆角不超过 8px。

### 5.2 任务节点

用户确认工具参数后，在源节点右侧创建任务节点并建立连线。任务节点在同一空间位置展示以下状态：

- 等待中
- 生成中：进度条和百分比
- 已完成：自动替换为结果缩略图
- 失败：错误摘要、重试和查看详情
- 已取消

任务节点创建时同步进行用量预留。完成后按实际用量结算，失败或取消时释放未使用额度。

### 5.3 结果节点

一次任务可产生 1 至 4 个结果。多个结果在源节点右侧按横向组排列，共享一条任务连线和批次标签。任意结果都可继续作为新分支的来源。

结果节点包含版本、审核状态和下载门禁。仅“审核通过”的结果显示下载动作；待审核、驳回和生成中状态均不可下载。

### 5.4 连线语义

- 实线：已完成的来源关系。
- 动态蓝色线：执行中的任务关系。
- 灰色虚线：失败或取消的任务关系。
- 连线只表达版本血缘，不允许用户连接出循环依赖。

## 6. 工具与上下文面板

### 6.1 悬浮工具栏

工作台左上方提供竖向图标工具栏：

- 生成
- 融图
- 快速视角
- 定向光
- 去除
- 抠图
- 扩图
- 超分

按钮仅显示图标，悬停时显示中文名称和用途。未选中节点时，需要输入图片的工具呈禁用状态。

### 6.2 上下文参数面板

点击工具后，在选中节点右侧打开深色浮动面板。面板具备标题、关闭按钮、参数控件、预计用量和蓝色主操作按钮。面板位置应避开视口边缘，空间不足时自动翻转到节点左侧。

参数遵循现有 MVP 能力：

- 生成：提示词、数量、比例、质量。
- 融图：参考素材、融合强度、提示词。
- 快速视角：水平角度、俯仰角度、距离。
- 定向光：光源方向、强度、色温；控制点覆盖在图片上。
- 去除：局部遮罩和提示词。
- 抠图：背景处理和边缘精度。
- 扩图：目标比例、缩放和补充提示词；裁切框覆盖在图片上。
- 超分：倍率和细节增强等级。

提交后面板可自动收起，画布聚焦新建任务节点。

### 6.3 素材选择浮层

需要参考图时打开素材选择浮层，包含搜索、素材分组和缩略图网格。用户选择后在参数面板内以缩略图槽位展示，并可替换或移除。

## 7. 中文化规则

所有用户可见界面使用简体中文。核心术语统一如下：

| 英文概念 | 中文显示 |
| --- | --- |
| Dashboard | 首页 |
| Projects | 项目 |
| Image Studio | 图片工作台 |
| Assets | 素材库 |
| Reviews | 审核 |
| Usage | 用量 |
| Admin | 企业管理 |
| Generate | 生成 |
| Blend | 融图 |
| Quick Angle | 快速视角 |
| Directional Light | 定向光 |
| Expand | 扩图 |
| Upscale | 超分 |

任务状态、错误提示、空状态、按钮、工具提示和表单标签均不得残留英文。

## 8. 视觉规范

### 8.1 色彩

- 画布背景：`#08090B`
- 主面板背景：`#15161A`
- 浮层背景：`#1C1D22`
- 分隔线与边框：`#34363D`
- 主文字：`#F3F4F6`
- 次文字：`#9297A1`
- 品牌操作蓝：`#2F80FF`
- 成功：`#25B47E`
- 警告：`#E7A23B`
- 错误：`#E45757`

图片内容承担主要色彩，不使用大面积紫色、米色、棕色或蓝灰单色主题，不使用渐变球、光斑和营销式装饰。

### 8.2 排版与密度

- 工具界面文字使用 12px 至 14px。
- 节点标题使用 12px，中等字重。
- 页面不使用超大标题。
- 字间距保持 `0`，字号不随视口宽度缩放。
- 图标按钮提供稳定的 32px 或 36px 点击区域。

### 8.3 层级

- 画布节点：基础层。
- 节点快捷动作和工具栏：操作层。
- 参数面板、素材选择器：浮层。
- 确认弹窗和错误详情：模态层。

所有浮层使用细边框和克制阴影，不套用卡片中的卡片。

## 9. 状态与数据流

### 9.1 前端状态扩展

现有场景数据增加持久化画布字段：

```ts
type CanvasPosition = {
  x: number;
  y: number;
};

type SceneNodeState = {
  sceneId: string;
  position: CanvasPosition;
  width?: number;
  selected?: boolean;
};
```

工具面板状态独立于领域数据：

```ts
type WorkbenchUIState = {
  activeTool: ToolKind | null;
  selectedSceneIds: string[];
  focusedSceneId: string | null;
  isSceneRailCollapsed: boolean;
  isTaskTrayOpen: boolean;
};
```

### 9.2 操作流程

1. 用户选中源节点。
2. 用户选择工具并填写参数。
3. 前端校验素材、提示词、权限和剩余额度。
4. 创建任务记录、预留用量，并在源节点旁插入任务节点。
5. 模拟异步任务更新进度。
6. 成功时生成结果节点、结算用量并进入待审核状态。
7. 失败时保留失败节点、释放额度并提供重试。
8. 审核通过后开放下载。

### 9.3 位置更新

React Flow 的 `onNodesChange` 和 `onNodeDragStop` 写回场景位置。任务和结果首次创建时根据源节点位置自动排布；用户拖动后不再被自动布局覆盖。

## 10. 错误与边界处理

- 无选中节点：相关工具禁用并显示中文工具提示。
- 缺少参考素材：提交按钮禁用，字段内显示原因。
- 额度不足：不创建任务节点，打开用量说明。
- 任务失败：失败节点留在画布中，可查看原因、重试或删除。
- 节点被删除：仅删除当前版本；存在子分支时需要二次确认。
- 自动保存失败：顶栏显示失败状态，保留本地操作，不阻塞继续编辑。
- 超小视口：桌面编辑区显示“请在桌面端编辑”，并提供任务与审核入口。

## 11. 响应式策略

- `>= 1200px`：完整全局导航、场景栏、画布和上下文浮层。
- `768px - 1199px`：全局导航折叠，场景栏默认收起，保留画布编辑。
- `< 768px`：不提供节点拖拽和参数编辑，仅显示项目预览、任务列表和审核操作。

固定格式控件使用明确尺寸或比例约束，避免节点加载、状态变化或长中文文案导致布局跳动。

## 12. 组件边界

建议将现有单体工作台拆分为以下组件：

- `WorkbenchShell`：顶栏、全局导航和工作区框架。
- `SceneRail`：场景树与素材库。
- `CanvasWorkbench`：React Flow 画布、节点和连线。
- `SceneNode`：源图、任务和结果的统一节点外壳。
- `ToolPalette`：悬浮工具入口。
- `ContextToolPanel`：按工具渲染参数控件。
- `AssetPicker`：素材搜索与选择。
- `TaskTray`：跨画布任务队列。
- `CanvasControls`：缩放、适配和小地图。

领域规则继续放在 `src/domain.ts`，组件不直接复制用量、审核和下载判断逻辑。

## 13. 测试与验收

### 13.1 单元测试

- 节点拖拽后位置写回状态。
- 新任务按源节点位置生成默认坐标。
- 用户手动移动后的节点不被重新布局覆盖。
- 各工具的必填参数和额度校验。
- 成功、失败和取消时的用量预留与结算。
- 审核通过前禁止下载，审核通过后允许下载。

### 13.2 组件测试

- 全局导航、场景栏、工具栏和参数面板均显示中文。
- 未选中节点时工具禁用。
- 选择工具后参数面板出现在工作区并可关闭。
- 生成中、失败、待审核和已通过节点状态正确展示。
- 素材可以拖入画布创建节点。

### 13.3 浏览器验收

- 在 1440px、1280px、1024px 和 375px 视口截图检查。
- 验证画布可平移、缩放、拖拽，节点位置不会跳回。
- 验证浮层不会超出视口或遮挡主要操作。
- 验证长中文项目名、节点名和错误信息不溢出。
- 验证生产构建通过，浏览器控制台无错误。
- 验证 `http://127.0.0.1:5173/` 可正常访问并保持开发服务器运行。

## 14. 完成标准

满足以下条件即视为本轮改版完成：

- 图片工作台成为深色、画布优先的节点式界面。
- 节点可自由拖拽并保留位置。
- 工具以悬浮工具栏和上下文面板工作。
- 生成结果在画布中形成清晰分支和连线。
- 所有主要界面与状态均为中文。
- 现有领域规则、测试和生产构建不发生回归。
- 浏览器在桌面和移动视口完成视觉与交互验收。
