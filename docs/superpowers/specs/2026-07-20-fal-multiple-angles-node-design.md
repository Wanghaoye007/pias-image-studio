# PIAS Fal 多角度节点接入设计

## 目标

将现有 `angle` /「快速视角」工具升级为真实可用的「多角度」功能节点，接入 Fal 模型 `fal-ai/qwen-image-edit-2509-lora-gallery/multiple-angles`。保留 `angle` 工具 ID、历史场景链路和任务领域模型，不新增重复节点；用户在画布中设置角度后，任务通过本地服务端代理提交到 Fal，完成后以真实图片结果节点回到画布。

## 已选方案与备选方案

采用“现有节点兼容升级 + Vite 服务端队列代理”。该方案保持当前画布交互和数据兼容，同时确保 Fal Key 只存在于 Node.js 服务端进程。

- 采用方案：保留 `angle` ID，将名称更新为「多角度」，前端调用本地 `/api/fal/multiple-angles` 队列代理。改动集中，历史数据兼容，适合当前本地 Vite 服务。
- 备选一：新增独立 `multiple-angles` 工具。会造成「快速视角」和「多角度」功能重复，并要求迁移工具 ID，因此不采用。
- 备选二：浏览器直接调用 Fal。实现更少，但会把 Fal Key 暴露给前端和浏览器网络面板，因此不采用。

## 范围

本批实现以下能力：

- 工具名称、参数面板、任务卡和结果标题统一显示「多角度」。
- 单个任务使用当前 Scene 或派生 Result 的图片作为主输入。
- 支持水平旋转、镜头推进、垂直视角、广角镜头、画面比例和 1 至 4 张候选结果。
- Fal 任务使用真实队列提交、状态轮询和结果读取，不再走现有模拟定时器。
- Fal 返回的图片 URL、尺寸、Seed、模型 ID 和请求 ID 进入任务或结果元数据，供详情和审计追踪。
- 失败时任务进入失败态并释放预留额度；重试会创建新任务并重新提交。
- Key 从服务端环境变量或本地 Key 文件读取，不写入仓库、不返回浏览器、不出现在日志中。

本批不实现 Precise Angle 的 8 帧旋转预览、自动 SKU 相似度模型、Fal 计费到账单同步和生产对象存储迁移。当前工作台额度仍作为产品内演示额度结算，Fal 账户的真实计费由 Fal 平台记录。

## Fal 接口映射

模型 ID 固定为 `fal-ai/qwen-image-edit-2509-lora-gallery/multiple-angles`。请求只发送官方输入字段：

| 工作台字段 | Fal 字段 | 规则 |
| --- | --- | --- |
| 当前 Scene / Result 图片 | `image_urls` | 至少一张；本地资源转为 Data URI，公网图片保留 HTTPS URL |
| 水平旋转 | `rotate_right_left` | `-180` 至 `180`；正值向左、负值向右 |
| 镜头推进 | `move_forward` | `0` 至 `10`；`0` 不推进，`10` 为近景 |
| 垂直视角 `verticalView` | `vertical_angle` | `-1` 至 `1`；`-1` 俯视，`0` 平视，`1` 仰视 |
| 广角镜头 | `wide_angle_lens` | 布尔值 |
| 画面比例 | `image_size` | `1:1`、`4:5`、`3:4`、`16:9` 映射为固定宽高对象 |
| 产出数量 | `num_images` | 整数 `1` 至 `4` |

其余模型参数使用稳定默认值：`guidance_scale=1`、`num_inference_steps=6`、`acceleration="regular"`、`negative_prompt=" "`、`enable_safety_checker=true`、`output_format="png"`、`lora_scale=1.25`。界面不展示当前不需要用户调整的模型参数。

旧的 `horizontalAngle` 参数继续沿用。原 `distance` 和角度制 `verticalAngle` 仅作为旧任务快照保留；新任务分别写入 `moveForward` 和 `verticalView`，从字段层面消除新旧垂直角度语义歧义，不修改已经结算的历史任务。

## 服务端架构

新增一个职责单一的 Fal 多角度服务模块和一个 Vite 中间件插件：

- 服务模块负责读取并解析 Key、校验请求、构造 Fal 输入、提交队列、查询状态、读取结果和取消任务。
- Vite 插件同时挂载到开发服务和预览服务，暴露同源 JSON API；前端永远不接触 Fal 凭证。
- 本地开发的 Key 读取优先级为 `FAL_KEY`、`FAL_KEY_FILE` 指向的文件、默认文件 `~/Desktop/key.md`；生产服务禁用默认文件回退。文件可包含裸 Key 或 `FAL_KEY=...`，解析后只保存在服务端内存。

本地 API 契约：

- `POST /api/fal/multiple-angles/jobs`：校验并提交，返回 `{ requestId }`。
- `GET /api/fal/multiple-angles/jobs/:requestId/status`：返回标准化的 `queued | running | completed`、日志和可选进度。
- `GET /api/fal/multiple-angles/jobs/:requestId/result`：返回 `{ images, seed }`。
- `DELETE /api/fal/multiple-angles/jobs/:requestId`：请求取消；本地已结算任务保持幂等。

所有 API 错误返回中文安全消息和稳定错误代码。上游响应、请求头和 Key 不写入浏览器响应或控制台。

## 前端任务流

用户提交 `angle` 任务后，领域层仍先创建本地 Job 并冻结额度。执行器根据 `profileId` 分流：

- 非 `angle` 任务继续使用现有演示定时器，避免扩大本次接入范围。
- `angle` 任务读取任务输入快照对应的 Scene 图片，将本地图片转为 Data URI，调用本地提交 API，并把 Fal `requestId` 写入 Job。
- 前端按短间隔轮询状态，将 Fal 队列阶段映射为本地进度；完成后读取结果并调用真实结果结算函数。
- 组件卸载、任务取消或任务进入终态时停止轮询，避免重复结算。

结果结算函数接受真实图片列表，而不是从演示图片池取图。每个 Result 保存 Fal URL、宽高、Seed、模型 ID、请求 ID 和当前角度参数；结果默认仍为草稿、未采用、非主图，符合 Quick Angle 的人工检查要求。

## 节点交互

`angle` 节点在工具栏、拖拽创建面板、上下文参数面板、任务卡和结果标题中显示「多角度」。参数面板使用模型原生语义：

- 水平旋转滑杆：`-180°` 至 `180°`。
- 镜头推进滑杆：`0` 至 `10`。
- 垂直视角滑杆：`-1` 至 `1`，辅助标签展示“俯视 / 平视 / 仰视”。
- 广角镜头开关：默认关闭。
- 产出数量沿用通用数量控件，限制为 1 至 4。
- 面板持续显示“模型会推断不可见区域，结果需人工复核”的风险提示。

由于该 Fal 接口没有 prompt 字段，多角度节点不发送创作描述。其他节点的创作描述交互保持不变。

## 数据模型与审计

`GenerationJob` 增加可选的外部执行信息：供应商、模型 ID、请求 ID。`Result` 增加可选的生成元数据：Seed、模型 ID、请求 ID 和目标视角参数。可选字段保证当前演示任务和已有测试数据无需迁移。

审计事件继续使用现有 `job.created`、`job.succeeded`、`job.failed` 和 `job.canceled`，并在可用时附带供应商、模型 ID 和 Fal 请求 ID。Key、Data URI 和完整上游日志禁止进入审计详情。

## 错误与取消

- Key 缺失或格式无效：提交前失败，显示“Fal 服务凭证未配置”，并释放本地预留额度。
- 输入图片无法读取：任务失败，显示“无法读取多角度输入图片”。
- Fal 参数校验失败：任务失败并显示可操作的中文参数错误，不自动重试。
- Fal 队列或网络临时失败：任务失败并保留“重试”入口；重试创建新的本地 Job 和 Fal 请求。
- 安全检查无结果：按“任务未生成可用结果”失败，不创建空结果节点。
- 用户取消：停止轮询并尽力取消 Fal 队列；无论上游取消是否及时，本地任务只结算一次并释放预留额度。

## 测试与验收

采用测试先行，覆盖参数映射、Key 解析、代理路由、真实结果结算和工作台交互。Fal SDK 在自动化测试中使用边界替身，不进行付费调用；最终验收单独使用真实 Key 生成一张图片。

验收条件：

1. 画布只出现一个「多角度」功能，内部工具 ID 仍为 `angle`。
2. 调整四项模型参数并提交后，本地请求体与 Fal 官方字段和数值范围一致。
3. 浏览器源码、请求和日志中不出现 Fal Key；仓库没有新增凭证文件。
4. Fal 队列状态能驱动任务卡从等待、处理中到完成或失败，且不会重复结算。
5. 成功任务生成真实图片结果节点，详情可看到尺寸、Seed、模型 ID 和请求 ID。
6. 失败或取消任务不创建结果，冻结额度归零，重试可提交新请求。
7. 全量 Vitest、TypeScript 构建通过；桌面与移动端截图中参数面板、任务节点和结果节点无重叠或溢出。
8. 本地服务重启后 `http://127.0.0.1:5173/` 可访问，并完成一次真实单图多角度生成验证。
